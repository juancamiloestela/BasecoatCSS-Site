Components in Basecoat also aim to be as simple as possible so that you can skin them or configure them as needed. However using them out of the box will give a beautiful minimalistic result.

CheckboxJS

CheckboxJS, like SelectJS, aims to skin the checkbox and radio button controls as efficiently as possible. This allows us to make the inputs behave and maintain correct typographical dimensions and relation to surrounding elements, while keeping accessibility, tabbed navigation and keyboard controls robust on all device/browser combinations.

SelectJS

Selectjs offers a clean and minimal way to skin <select> elements. It focuses on retaining the most out of the native OS control so that accessibility, tabbed navigation, keyboard controls and device specific options list are consistent. This allows us to have <select> elements that maintain beautiful typography and styles with surrounding elements while ensuring the control works as it should in the most efficient manner.

PeekJS

Peekjs gives you a full fledged off canvas menu that is triggered by css classes and also provides touch controlled animations on mobile devices. The effects have been abstracted so that you can choose the way the off canvas content is shown by applying the desired coat.

Try it!
Toggle Left Menu Toggle Right Menu
Usage

To use peekjs, your layout must have a specific structure. In your <body> you must set the class that defines which effect to use.


<body class="peek-effect-door">

Right inside the <body> tag you must define one or two containers that will have the off canvas content. These will hold your off canvas content.


<body class="peek-effect-door">
    <div class="peek-left">
        <a href="#" class="peek-toggle-left">Close</a>
        This is your off canvas content shown on the left
    </div>

    <div class="peek-right">
        <a href="#" class="peek-toggle-right">Close</a>
        This is your off canvas content shown on the right
    </div>

At the same level of the peek left and right containers you must now define a peek-container. This element will crop the page during peek events. This allows us to have content inside it that behaves as if it had position fixed, but it won't screw up when filling forms on iOS. In this example we have set up a fixed header. If we wanted the header to scroll with the rest of the content we would move it into the next required element, the container with class peek-scroll-mask. Inside this container you can place all the content of your page.


<body class="peek-effect-door">
    <div class="peek-left">
        <a href="#" class="peek-toggle-left">Close</a>
        This is your off canvas content shown on the left
    </div>

    <div class="peek-right">
        <a href="#" class="peek-toggle-right">Close</a>
        This is your off canvas content shown on the right
    </div>

    <div class="peek-container">
        Your page
    </div>

At the end of the document you can optionally set the curtain element that will cover the page content when the user is peeking off canvas. You can create it if you need a custom curtain, if not, peekjs will create a default one.


    <div class="peek-overlay"></div>
</body>
Peek Controls

To peek into the off canvas containers you can set on any element the class="peek-toggle-right/left" and when these elements are clicked, the content will be shown. To control it via js you can call the exposed Peekjs methods.


<a href="#" class="peek-toggle-right">Toggle Menu</a>
<a href="#" onclick="Peekjs.openRight();">Open Menu</a>
Method Description
Peekjs.openLeft(); Opens the left container
Peekjs.openRight(); Opens the right container
Peekjs.closeLeft(); Closes the left container
Peekjs.closeRight(); Closes the right container
Peekjs.toggleLeft(); Toggles the left container
Peekjs.toggleRight(); Toggles the right container
Peekjs.closeAll(); Closes all containers
Hide Content on Peek

To hide elements (eg. a floating header) when the user peeks, just give the elements you wish to hide the class class="peek-hide-on-peek".

Customizing the effect

Peekjs comes with 4 effects, to define an effect you do it via a class on the <body> tag. The effects are 'peek-effect-door', 'peek-effect-minimize', 'peek-effect-lay-down' and no class for the default effect.

Open Right Default Open Right Door Open Right Minimize Open Right Lay Down

When the user peeks, the <body> gets a set of classes that can be used to design your preferred transition. When peeking via a button click, or a js method. The body will get a class of peek-open-left/right. When peeking on a mobile device via drag, you will get two additional classes, peek-transitioning and peek-opening/closing-transition.

	<li>Item 6</li>

<style>
	/* Default state */
	.sequenced-list li{
		opacity: 0;
		-ms-transition: opacity 300ms ease-in;
		transition: opacity 300ms ease-in;
	}
</style>

<script>
	function runSequenceListDemo(){
		$('.sequenced-list li').sequence({
			offset: 600, // ms between items
			properties: {
				opacity: 1
			},
			setup: function(elements){
				// clean up
				$(elements).removeAttr('style');
			}
		});
	}
</script>

You can also sequence the addition or removal of classes, instead of css properties. Keep in mind that sequencing many elements via classes instead of properties directly will be slower because the browser has to recalculate styles every time a class is set.

A sequenced list by classnames
Run

Hi, How are you today? 


<a href="#" class="button" onclick="runSequencePhraseDemo();">Run</a>
<p class="sequenced-phrase">
	<span>Hi,</span>
	<span>How </span>
	<span class="ugly">are </span>
	<span>you </span>
	<span>today?</span>
</p>

<style>
	/* Default state */
	.sequenced-phrase span{
		opacity: 0;
		transform: translateY(30px);
		-ms-transition: all 300ms ease-in;
		transition: all 300ms ease-in;
	}

	.sequenced-phrase span.ugly{
		background-color: green;
		opacity: 0.5;
		color: #fff;
	}

	/* Visible states */
	.sequenced-phrase span.show{
		opacity: 1;
		transform: none;
	}
	.sequenced-phrase span.red{
		color:red;
	}
</style>

<script>
	function runSequencePhraseDemo(){
		$('.sequenced-phrase span').sequence({
			offset: 1000, // ms between items
			add: 'show red', // adds the class show and red
			remove: 'ugly', // removes the class 'ugly'
			setup: function(elements){
				$(elements).removeClass('show red');
				alert('Starting phrase sequence');
			},
			done: function(elements){
				alert('Finished phrase sequence');
			}
		});
	}
</script>

If you don't have jQuery (or a replacement), you can still use sequencejs. But keep in mind that when setting css properties directly, you must set the vendor prefixed properties also as if you were doing a element.style.PROPERTY

			}).play();

DismissableJS

The dismissable plugin listens for clicks on any element with class .dismiss that is inside an element with class .dismissable. When a click is detected, the element with class .dismissable will fade out and then removed from the dom. There is no initialization or configuration, the plugin is as simple as possible and autostarts when the om is ready. It listens for clicks on the body of the document so if a the dismissable element is inside a container that stops propagation, nothing will happen.

Loaders

Showing the user that content is loading is a must for a rich user experience. Basecoat provides a simple loading bubble that can be used inline with text or as a prominent loading message. The loader bubble is also proportional to the current font size so it will be perfectly aligned with your content. You can also change the color of the loading bubble by setting its background color.
