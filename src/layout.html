Layout elements in Basecoat aim to be as simple as possible. Their only purpose is to position content as desired en all screens and resolutions while keeping a coherent visual style that protects typographical balance.

These are the only elements that are responsive by default. Any additional responsive behavior is considered custom and should be implemented as a coat. Basecoat defines media queries for mobile first (0 to 30em), tablet (31em to 50em) and desktop (50em and up). Breakpoints are defined in ems and not in pixels because it is the font size that will determine if the content should be reorganized or not. You could have a 320px screen with a large font size which will make the content break in many lines or you could have the same screen with a tiny font which would look like a zoomed out version of a desktop page. In Basecoat it is the content and the screen that define if it should reorganize itself, not the screen alone.

Dividers

Showing the user that content is loading is a must for a rich user experience. Basecoat provides a simple loading bubble that can be used inline with text or as a prominent loading message. The loader bubble is also proportional to the current font size so it will be perfectly aligned with your content. You can also change the color of the loading bubble by setting its background color.

Grids

In Basecoat, grids are not restricted to the famous 12 column grid. In Basecoat you choose how many columns your grid should have from 2 to 16. Any element can become a grid, a div, span you name it. Grids also have no margins, gutters are achieved via cell paddings which gives you more flexibility when styling your layouts because it is the inner content who defines it's presentation, the grid only provides a space for the content to exist.

Because we can choose the number of columns to use, the syntax is a little different than what you might be used to. It is much more intuitive. Lets say you need to divide a container into five equal columns. We would need a five column grid, then that element's class will be five columns, that simple. You can replace the word "five" for any number between "two" and "sixteen".

Layouts

Layouts are simple layout structures that can be used on any part of your page and can be nested to achieve more complex layouts. They come in two flavors, row based layouts and column based layouts. As with grids, any element can be a layout, but beware! layouts use flexbox so older browsers will blow up.
